# 特定健診WEB問診システム

ヘルスケアITの現場で10年以上携わってきた経験をもとに、
「実務で本当に使えるレベル」を意識して開発したWEB問診システムです。

紙の問診票をそのままデジタル化するのではなく、
受診者が当日慌てずスムーズに回答できるUXと、
スタッフが余計な操作なく管理できる画面設計にこだわりました。

**デモ URL:** https://web-kenshin-app.vercel.app

---

## 想定ユーザー

| ユーザー | 利用シーン |
|---|---|
| 受診者 | 来院前にスマホ・タブレットでQRコードを読み取り、問診に回答 |
| クリニックスタッフ | 受診者の登録・管理、回答状況の確認、集計レポートの閲覧 |

---

## 主な機能

### 受診者向け
- QRコードからワンクリックでアクセス（ログイン不要）
- ウィザード形式（1問ずつ表示）＋進捗バー表示
- 4言語対応（日本語・英語・中国語簡体字・繁体字）
- 回答の保存と再編集に対応

### スタッフ向け（管理画面）
- 受診者一括登録（CSVインポート）
- 回答状況の一覧確認（未回答 / 一部回答 / 回答済みバッジ）
- 回答内容の詳細閲覧・CSV出力
- 分析グラフ（回答率・カテゴリ別・設問別）
- QRコード一括発行（4件発行レイアウト対応）
- 問診設問の追加・編集・並び順変更・有効無効切替
- スタッフアカウント管理（追加・削除・パスワード変更）
- セキュリティログ（ブルートフォース対策・セッションタイムアウト）

## こだわった点

**受診者UX**
1問ずつ表示するウィザード形式は、高齢の受診者でも迷わないよう意識しました。
選択肢はラジオボタンではなくボタンカード形式にして、タブレットでの操作感を改善しています。

**色覚多様性への対応**
医療現場では色覚特性を持つスタッフ・受診者への配慮が必要です。
赤/緑の組み合わせを避け、エラーはオレンジ・成功は青系で区別しました。

**レスポンシブ設計**
スタッフ画面はPCでテーブル形式、スマホではカード形式に自動切替。
現場ではタブレットやスマホで操作するケースも多いため、タップしやすいボタンサイズにも気を配りました。

- アクセント：ティール `#00B4D8`

## ローカル起動手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/KaNaHiRi/web-kenshin-app.git
cd web-kenshin-app

# 2. 依存パッケージをインストール
npm install

# 3. 環境変数を設定（.env.localを作成）
cp .env.example .env.local
# DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL を編集

# 4. DBマイグレーション & シードデータ投入
npx prisma migrate deploy
npx prisma db seed

# 5. 開発サーバー起動
npm run dev
```
